Travelling Salesman Problem

Overview

The travelling salesman problem (TSP) is an algorithmic problem tasked with finding the shortest route between a list of cities that must be visited exactly once and return to the starting city. It is a famous algorithmic problem in the fields of computer science and operations research because it is easy to state yet tough to solve. The complexity of obtaining a best route for the problem will also grow exponentially as more cities are added. For example, with 10 cities, there are more than 300,000 different routes. However, with 15 cities, the number of possible route increases to more than 87 billion. Hence, it is impractical for computer to calculate every possible route to find out the shortest route. We would need other methods or algorithms to find a good solution for TSP.

There also exists several variations for TSP. For our website, TSP will be implemented to minimise route distance with the assumption of the starting city is always fixed to the same city.

Representation of TSP in Computer

a) Weighted Graph

TSP can be represented in computer by using weighted graph, where the graph’s vertices represent the cities, the graph’s edges represent the path, and the weight of an edge represent the distance.


b) Adjacency Matrix

Adjacency Matrix is a square matrix used to represent a graph in graph theory and computer science.

An entry Mij in the adjacency matrix representation shows the weight from Vi to Vj.

Tackling TSP with Optimisation Algorithms

Optimisation is the process of choosing inputs that could increase the efficiency of a solution, or obtain best possible outputs. However, the optimal results are often not guaranteed. It also often involves in either maximising or minimising a value. In this context, we try to minimise the distance for the route.

The optimistaion algorithms can be further divided into two types, namely heuristic method and metaheuristic method. The heuristic method is not generic and it is a problem specific approximating solution. In other words, it solves the problem by using some of the features of the given problems. On the other hand, metaheuristic is more generic and not problem dependant. It can be applied to solve a problem despite without having knowledge about the problem.
